Дополнительные характеристики

Авторизация
Тип авторизации
Указывается, если нужно парсить под конкретной учетной записью, и не запрашивать логин/пароль у пользователя
Страница авторизации
Указывается, если форма авторизации не на главной странице.
XPath формы
авторизации
Определяется автоматически как первая POST форма с контролом для ввода пароля. При необходимости можно указать вручную.
Страница
POST-запроса
Определяется автоматически по форме, оставлено для поддержки старых настроек. При необходимости можно указать вручную.
Шаблон тела
POST-запроса
Определяется автоматически по форме. При необходимости можно указать вручную выражение вида username={login}&password={pass}
Дополнительные заголовки
запроса
Указывается если необходимо передать какие-либо дополнительные заголовки. Например, {"Accept": "application/json", "Content-Type": "application/json"}
Тип проверки
авторизованности
Определяется автоматически как отсутствие формы авторизации. При необходимости можно указать вручную.
Текст пользователю
Указав логин и пароль от сайта, Вы сможете загружать оптовые цены.
(Указывается выделенная в тексте часть, она же - значение по умолчанию, если ничего не указано.)
Включено, если на сайте отображаются одни и те же данные под разными логинами, например, цена для всех оптовиков одинаковая.

Кеширование
- не используется при тестовой загрузке (?test=1)
Область кеширования
Область кеширования не должна содержать меняющиеся данные: время загрузки страницы, комментарии, голосования... Если XPATH не указан - кешируется вся страница.
